# =============================================================================
# HaWoR — CAM-SPACE HANDS ONLY — Blackwell requirements (CUDA 12.8)
#
# Target: server with 8x NVIDIA RTX PRO 6000 Blackwell (sm_120 / compute 12.0).
# Scope:  ONLY the camera-space hand-trajectory stages used by
#         run_mv1_noinfiller.py  ->  build frames -> detect+track (YOLO+Molmo)
#         -> motion estimation (cam_space) -> no-infiller cam render.
#         SLAM (DROID-SLAM / lietorch / eigen), the neural infiller, and the
#         world-space vis are NOT part of this path, so their CUDA builds are
#         intentionally omitted.
#
# Box prerequisites (NOT pip-installable):
#   - NVIDIA driver >= 570 (Blackwell / CUDA 12.8).
#   - CUDA 12.8 toolkit with nvcc (e.g. /usr/local/cuda-12.8). Needed ONLY to
#     compile pytorch3d for sm_120 (see step 2). Set CUDA_HOME to it.
#   - Python 3.10  (the torch/torch-stack wheels pinned below are cp310).
#
# Install (in this order):
#   python -m pip install --upgrade pip
#   # 1) everything below, incl. the cu128 torch stack:
#   python -m pip install -r requirements_blackwell.txt
#   # 2) pytorch3d — must COMPILE for Blackwell (no reliable prebuilt for
#   #    pt2.8+cu128); build against the freshly installed torch, targeting sm_120:
#   TORCH_CUDA_ARCH_LIST="12.0" CUDA_HOME=/usr/local/cuda-12.8 \
#     python -m pip install --no-build-isolation \
#       "git+https://github.com/facebookresearch/pytorch3d.git@stable"
#
# 8-GPU note: this pipeline is single-GPU per video. Fan out one video per GPU
#   with CUDA_VISIBLE_DEVICES=<0..7>; no code/dependency change is needed for it.
# =============================================================================
